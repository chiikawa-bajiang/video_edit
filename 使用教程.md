## 一、启动软件

### 方式 A：开发模式启动（当前使用）

1. 打开 PowerShell，进入项目目录：

```powershell
cd C:\Users\milktea\Desktop\一刀
```

2. 启动：

```powershell
npm start
```

稍等几秒，会弹出深色窗口「一刀 · 视频处理工作台」。

### 方式 B：双击 exe 启动（打包后）

打包后进入 `dist\一刀视频工作台-win32-x64\`，双击 `一刀视频工作台.exe` 即可，无需 Node 环境。

## 二、界面总览

软件左侧是**功能模块导航**，目前有三个模块：

| 图标 | 模块 | 用途 |
|---|---|---|
| 🎬 | **FFmpeg 视频处理** | 对单条视频做精细调整（改编码/帧率/码率/分辨率/调色/锐化/模糊/裁剪等） |
| 🗂️ | **批量生成** | 基于一条原视频，批量产出多条"微调去重"的竖屏短视频 |
| 📦 | **千川素材交付** | 完整流水线：导入→切片→套模板→渲染→质检→导出标准交付包 |

右上角的状态标签会显示引擎状态（就绪 / 处理中）。

## 三、模块一：FFmpeg 视频处理（单条精修）

**适合场景**：手上有一条视频，想改个参数 / 转个竖屏 / 调个色 / 剪一段。

### 操作步骤

1. 点 **「选择视频…」**，选一个视频文件。
   - 选好后上方会显示：时长、分辨率、帧率、编码、码率、大小。

2. 顶部 **「📱 竖屏 / 手机尺寸」** 面板（默认开启）：
   - **输出为竖屏手机尺寸**：勾选后输出适配抖音的竖屏画面。
   - **画布尺寸**：1080×1920（抖音 9:16）/ 720×1280 / 1080×1350 / 1080×1080。
   - **适配方式**：
     - `填充裁剪`——放大铺满，最常用；
     - `模糊背景`——画面居中，上下留白处用模糊画面补满（抖音常见风格）；
     - `黑边适配`——保留完整画面 + 上下黑边。
   - 不需要竖屏就取消勾选。

3. 四个参数面板按需调整：

   | 面板 | 可调项 |
   |---|---|
   | 编码 / 格式 | 视频编码（H.264/H.265/VP9…）、编码预设、码率、CRF 质量 |
   | 画面 / 尺寸 | 宽/高、常用分辨率预设、帧率 FPS |
   | 调色 / 滤镜 | 亮度、饱和度、锐化、模糊（拖滑块） |
   | 时间 / 裁剪 | **起始时间（第一帧从第几秒开始）**、截取时长、移除音频 |

4. 中间 **「命令预览」** 会实时显示即将执行的 ffmpeg 命令（看不懂可忽略）。

5. （可选）点 **「选择输出位置…」**。不选则自动存到源文件同目录，文件名带 `_一刀输出`。

6. 点 **「开始处理」**。
   - 下方进度条 + 日志实时更新。
   - 完成后自动弹出输出文件所在文件夹。

> 提示：点「重置参数」可一键恢复默认值。

## 四、模块二：批量生成（抖音去重微调）

**适合场景**：一条原视频，想批量产出很多条"看起来略有不同"的竖屏视频（短视频去重 / 多账号分发）。

### 通用设置（两种模式都要先做）

1. 点 **「选择原视频…」**。
2. 点 **「选择输出目录…」**（生成的视频都放这里）。
3. **「📱 输出尺寸」**：选竖屏画布和适配方式（同模块一）。

然后下方有两个标签页：**自动微调** 和 **CSV 导入**。

### 模式 A：自动微调（推荐）

1. 切到 **「🎲 自动微调」** 标签。
2. 设置：
   - **生成数量**：1～300 条。
   - **微调强度**：轻微 / 适中 / 较强。
   - **基础编码**：H.264 / H.265。
3. 勾选 **参与微调的维度**：亮度、对比度、饱和度、伽马、锐化、模糊、帧率、码率、变速（改时长）、轻微缩放、水平镜像、起始微剪。
4. （可选）勾「移除音频」。
5. 点 **「开始批量生成」**。
   - 后台逐条生成，显示**总进度 + 当前项进度 + 生成清单**。
   - 每条完成后可在清单点「打开」定位文件。
   - 输出文件名形如 `原名_抖音_001.mp4`。

> 原理：每条视频在原视频基础上做细微随机抖动，使每条都略有差异。

### 模式 B：CSV 导入（精确控制每条参数）

1. 切到 **「📄 CSV 导入」** 标签。
2. 点 **「⬇ 下载 CSV 模板」**，用 Excel 打开（中文表头，已含 3 行示例）。
3. 按需填写，**每一行 = 生成一条视频**。可用列：

   | 列名 | 说明 | 示例 |
   |---|---|---|
   | 文件名 | 输出文件名 | 示例_01 |
   | 编码 | libx264 / libx265 | libx264 |
   | 帧率 | 留空=不变 | 30 |
   | 码率 | 留空=自动 | 4000k |
   | 亮度 | -1 ~ 1 | 0.03 |
   | 对比度 | 1 附近 | 1.02 |
   | 饱和度 | 1 附近 | 1.05 |
   | 伽马 | 1 附近 | 1.00 |
   | 锐化 | 0 ~ 3 | 0.3 |
   | 模糊 | 0 ~ 20 | （留空） |
   | 变速 | 0.5 ~ 2，影响时长 | 1.01 |
   | 缩放 | 0 ~ 0.3，轻微放大 | 0.02 |
   | 水平镜像 | 1=镜像，0=否 | 0 |
   | 起始秒 | 从第几秒开始 | 0.2 |
   | 时长 | 截取时长 | （留空=到结尾） |
   | 移除音频 | 1=移除，0=保留 | 0 |
   | 竖屏模式 | fill / blur / fit | fill |
   | 竖屏尺寸 | 宽x高 | 1080x1920 |

   > 留空 = 该项不调整。

4. 保存 CSV，回到软件点 **「⬆ 选择 CSV 文件…」**，会显示行数和预览。
5. 点 **「开始批量生成」**，逐条按各自参数生成。

## 五、模块三：千川素材交付（流水线）

**适合场景**：把一批原片，批量加工成可直接交给运营/投手的"标准素材包"。

整个流程是 **5 个步骤**，按顺序点上方步骤标签即可（原来的 8 步已合并）。

### 步骤 1 · 准备（工作区 + 导入原片）

**① 工作区**
- 点 **「选择工作区目录…」**，挑一个空文件夹作为本项目的工作区。
- 软件会自动建好中文子目录（原片 / 切片 / 临时文件 / 成片 / 交付包 / 日志 / 进度数据），并加载已有进度。
- 下方显示当前原片/切片/模板/任务数量。

**② 导入原片**
- 点 **「选择视频（可多选）…」**，导入原片（当前流程按**单个原视频**设计，会用第一条作为切分对象）。
- 列表显示时长、分辨率、是否有声。

> 工作区就是这个项目的"档案柜"，所有中间文件和进度都存在里面，下次选同一目录可继续。

### 步骤 2 · 切片（手动切分）

- 上方是内置播放器，加载你导入的原片。
- 播放 / 拖动进度条到目标位置，点 **「设入点」**，再到结束位置点 **「设出点」**。
- 点 **「＋ 切一段」** 把这一段加入下方列表，自动命名 `切片1、切片2、切片3…`（顺序号自动维护）。
- 列表里每段显示 入点→出点 与时长，右侧 **✕** 可删除。
- 全部切完，点 **「生成切片」** 才真正切出文件并存入工作区 `切片` 目录。

> 播放器用 Chromium 解码，**H.264 + AAC 的 MP4 最稳**。若原片是 H.265/HEVC 等导致预览黑屏，先转成 H.264 MP4 再切。

### 步骤 3 · 模板（精确排序）

- 内置 3 个模板，点开可编辑：
  - **名称**
  - **比例**（9:16 / 1:1 / 16:9）
  - **目标时长(秒)**：`0` = 不限（成片＝所选切片之和）；填数值则裁剪封顶到该时长（如做 20s 视频就填 `20`）。
  - **变速选项**（逗号分隔，如 `1,1.05`，渲染时随机取一个）
  - **组合方式**：
    - **固定顺序**：所有成片都按下方「切片顺序」拼接（用 **+ 切片N** 添加、**↑ ↓** 排序、**✕** 移除，例如 `切片1 → 切片3 → 切片2`）。
    - **自由组合(随机排列)**：每条成片在「参与组合的切片」里**独立随机排列**，做到条条不同；可另选 **固定开头 / 固定结尾** 指定某片永远在首或尾，其余自由打乱；选「不固定」则首尾也参与随机。
  - **切片间转场**：切片衔接处的动态过渡，默认 **无（硬切）**。可选：
    - **淡入淡出 / 滑动 / 缩放 / 擦除 / 溶解**：固定该类风格（滑动/擦除会在上下左右等方向间随机）。
    - **每条随机**：每条成片、每个衔接处随机一种转场，画面更丰富，同时也是一个很强的**去重维度**。
    - **转场时长(秒)**：默认 `0.3`；会被自动限制在「不超过最短切片的一半」以内。
    - 注意：开启转场后，成片会因每个衔接处重叠约「转场时长」而**略变短**，且**渲染速度变慢**（转场需逐帧混合）。前期大批量建议先关，需要时再开。
- 改完自动保存。
- 模板管理（顶部按钮）：
  - **+ 新增模板** / 卡片上的 **复制 / 删除**（至少保留 1 个）
  - **导入 / 导出(JSON)**：备份或跨工作区复用。

> 想批量做「100 条 20s、切片自由排列组合」的视频：把模板**组合方式**设为「自由组合」、**目标时长**填 `20`、把要参与的切片都加进「参与组合的切片」，再到步骤4把**生成数量**填 100 即可；需要某片固定开场就设「固定开头」。
> 配合步骤4的 🎲 随机去重，即使偶尔出现相同排列，画面/编码参数也各不相同。

### 步骤 4 · 生产·渲染（生产计划 + 渲染）

**① 生产计划**
- 勾选要用的模板（可多选）、填 **生成数量**。
- 点 **「生成渲染任务」**：批量 N 条会在所选模板之间轮换。「固定顺序」模板每条按设定顺序拼接；「自由组合」模板每条**独立随机排列**切片（可固定首/尾）。统一套用比例/变速，目标时长>0 时裁剪封顶。
- 任务行会显示该条的实际顺序（如 `切片1→切片3→切片2`）、变速、封顶秒数；启用去重的任务行带 `🎲` 标记。
- 文件名规则：`BATCH001_TPL01_V001.mp4`。

**🎲 随机化去重（在「生成渲染任务」时生效）**

默认开启。作用是让**同一批每一条素材的参数都不同**，从而改变视频指纹，降低平台判重概率。

- **去重强度**：轻微 / 适中 / 较强（抖动幅度依次增大）。
- **参与去重的维度**（可勾选）：
  - **调色**：亮度、对比度、饱和度、伽马、色温、色调、色相、曝光、阴影高光。
  - **风格**：锐化或模糊（二选一）、颗粒、暗角、去噪。
  - **编码**：帧率、CRF、GOP、B帧、Profile（改变编码指纹）。
  - **微剪时长**：尾部随机裁掉一小段，改变时长与帧序列（保留开头钩子）。
  - **元数据**：清空原始元数据并写入自定义注释。
  - **几何**：轻微旋转 / 平移 / 缩放 / 透视（会改变画面观感，默认关闭，按需开启）。
  - **水平镜像**：约一半素材随机镜像（默认关闭）。
- 每条素材使用**独立随机种子**，种子会写入交付清单的 `dedupSeed` 列，便于追溯；完整去重参数保存在 `delivery-manifest.json` 的 `dedup` 字段。
- 关闭该开关即回到原行为（仅按模板比例/顺序/变速拼接）。

> 提示：几何与镜像去重最强，但也最容易被肉眼察觉，建议先用「调色 + 风格 + 编码 + 微剪 + 元数据」组合；需要更强去重时再逐步开启几何/镜像。

**② 音频设置**
- 保留原声 / 替换配乐 / 原声+配乐混音 / 静音，可选配乐文件并调音量。

**③ 渲染**
- **并发数量**：同时渲染几条（1 最稳，4 最快但吃 CPU）。
- 点 **「开始渲染」**，显示总进度 + 当前项进度 + 每条实时百分比与状态。
- **暂停 / 继续**：暂停后不再开新任务，进行中的会跑完；继续后接着排队。
- **停止**：立即终止全部。
- **重试失败项**：渲染结束后若有失败，只重跑失败的那几条。

> 配乐/混音失败时会自动转静音重试一次，尽量保证出片。

### 步骤 5 · 质检·交付（质检 + 交付）

**① 质检**
- 点 **「开始质检」**，自动检查每条成片：是否存在、体积、能否播放、时长、分辨率。
- 勾选 **深度质检** 后额外检测：**黑屏**、**整条静音 / 无音轨**、**视频过短**（以 ⚠ 标注为提醒，不强制判失败）。
- 列表显示通过/失败及原因。

**② 交付**

- 点 **「导出交付包」**，生成标准交付目录：

```text
交付包/2026-06-17_批次001/
  视频/              # 通过质检的成片
  封面/              # 每条自动截帧封面
  delivery-manifest.csv
  delivery-manifest.json
  qc-report.csv
  qc-report.json
  readme.txt         # 批次信息汇总
```

- 完成后点 **「打开交付目录」** 查看。

## 七、常见问题

**Q：处理失败 / 渲染失败怎么办？**
A：看模块下方的「处理日志」或任务清单里的失败原因。最常见是源文件损坏、路径异常、或参数极端（如码率填错）。

**Q：质检全部失败、提示"文件过小"？**
A：500KB 是最小体积阈值，正常视频不会触发。若用极简/纯色测试视频会误报，换真实视频即可。

**Q：能处理中文文件名 / 中文路径吗？**
A：可以，全程兼容中文和空格路径。

**Q：生成的视频没有声音？**
A：千川交付模块在「步骤4 · ② 音频设置」里选 **保留原声 / 替换配乐 / 混音 / 静音**；若某切片本身无音轨，保留原声会自动转静音。模块一、模块二默认保留音频（除非你勾了"移除音频"）。

**Q：处理速度慢？**
A：编码预设选更快的（如 ultrafast/veryfast），或降低分辨率/码率。H.265 比 H.264 慢但更小。

**Q：软件支持 Mac 吗？**
A：当前打包为 Windows x64。代码本身跨平台，需要时可另出 Mac 版。
