版本：V3.0  
使用对象：Cursor / Codex 多 Agent 直接开工  
最终交付：Windows 可执行 `exe` 桌面工具  
项目边界：只负责素材生产与交付，不负责千川投放数据追踪、ROI复盘、广告计划管理、投后生命周期管理。

## 2. 项目目标

开发一个本地桌面工具，将服装原始视频批量加工为可交付给运营/投手使用的素材包。

### 2.1 输入

- 原始视频文件：`mp4` / `mov`。
- 素材生产模板。
- 字幕/钩子文案。
- 可选背景音乐。
- 可选封面策略。

### 2.2 处理

- 导入原始视频。
- 读取视频元信息。
- 拆分镜头段。
- 组合素材结构。
- 添加字幕、封面、配乐、比例、节奏变化。
- 批量渲染。
- 基础质检。
- 导出标准素材包。

### 2.3 输出

```text
delivery/
  2026-06-17_batch001/
    videos/
      BATCH001_TPL01_HOOK01_V001.mp4
      BATCH001_TPL01_HOOK02_V002.mp4
    covers/
      BATCH001_TPL01_HOOK01_V001.jpg
    delivery-manifest.csv
    delivery-manifest.json
    qc-report.csv
    qc-report.json
    readme.txt
```

### 2.4 不做事项

- 不接入千川后台。
- 不导入投放数据。
- 不追踪广告计划。
- 不计算 CTR、CVR、ROI。
- 不做投放复盘看板。
- 不做素材投后生命周期管理。
- 不做预算调整。
- 不做素材效果预测。
- 不做 SKU 管理。

## 3. 技术路线

### 3.1 推荐技术栈

```text
npm + TypeScript + Electron + React + Vite + FFmpeg + FFprobe + 本地 JSON Manifest + electron-builder
```

### 3.2 为什么这样选

| 技术 | 用途 | 原因 |
|---|---|---|
| npm | 包管理和脚本调度 | Cursor/Codex 适配好，多 Agent 易协作 |
| TypeScript | 核心逻辑开发 | 类型约束清晰，适合多模块协作 |
| Electron | Windows 桌面端 | 前期即可有操作界面，便于观察和验收 |
| React | 桌面端界面 | 页面状态和组件拆分清晰 |
| Vite | 前端构建 | 启动快、配置轻 |
| FFmpeg | 视频切片、拼接、转码、字幕、配乐 | 视频处理核心依赖 |
| FFprobe | 视频元信息读取、质检 | 检查时长、分辨率、编码等 |
| JSON Manifest | 本地数据记录 | 不做投放追踪，前中期不强制数据库 |
| electron-builder | exe 打包 | 输出 Windows 可执行程序 |

### 3.3 与上一版不一致的点

| 项目 | 上一版 | 本版 | 原因 |
|---|---|---|---|
| 前期形态 | CLI 先行 | Electron 桌面 MVP 先行 | 你要求前期就要有操作界面可观察 |
| SKU 模块 | 有 SKU 导入/绑定 | 删除 | 当前没有 SKU 相关内容 |
| 数据存储 | JSON，可选 SQLite | 前中期统一 JSON Manifest | 项目只做素材交付，不做投后数据管理 |
| 看板 | 有 ROI/数据看板风险 | 删除 | 不管理追踪和复盘 |
| CLI | 主要前期入口 | 仅保留内部调试命令 | 主要入口改为桌面 UI |

## 12. 命名规则

素材文件名：

```text
BATCH001_TPL01_HOOK01_V001.mp4
```

含义：

- `BATCH001`：生产批次。
- `TPL01`：模板编号。
- `HOOK01`：钩子文案编号。
- `V001`：版本编号。

交付批次目录：

```text
YYYY-MM-DD_batch001
```

不得包含：

- SKU 字段。
- 千川计划字段。
- ROI 字段。

## 16. 质检标准

| 检查项 | 标准 | 失败处理 |
|---|---|---|
| 文件存在 | 输出 mp4 必须存在 | 标记 failed |
| 文件大小 | 大于最小阈值，例如 500KB | 标记 failed |
| 可播放 | ffprobe 可读取 | 标记 failed |
| 时长 | 与模板目标时长误差可控 | 标记 warning/failed |
| 分辨率 | 符合模板比例 | 标记 failed |
| 黑屏风险 | 首尾关键帧非纯黑 | 标记 warning |
| 音频 | 按模板要求保留/替换/混音 | 标记 warning |
| 命名 | 符合 BATCH_TPL_HOOK_V | 标记 failed |

质检输出：

```text
qc-report.csv
qc-report.json
```

## 22. 结论

本项目的开发重点是：

```text
桌面可操作
素材可生产
任务可观察
质量可检查
结果可交付
```

本项目不做：

```text
SKU
千川投放追踪
ROI复盘
广告计划管理
素材投后生命周期
```

完成前期后，团队应已经拥有一个可观察、可运行、可交付的桌面素材生产工具。完成中期后，工具应能以 Windows exe 形式交付给运营团队，稳定生产批量千川素材包。
